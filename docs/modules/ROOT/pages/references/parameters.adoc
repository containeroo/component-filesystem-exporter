= Parameters

This component is multi-instance. The parent key for all of the following parameters is the instance name you assign in your inventory.
For the default instance, that key is `filesystem_exporter`.

Example inventory snippet:

[source,yaml]
----
applications:
  - filesystem-exporter as archive

parameters:
  archive:
    namespace: syn-archive
    filesystem:
      path: /archive
    volume:
      nfs:
        server: nfs.example.internal
        path: /exports/archive
----

== `namespace`

[horizontal]
type:: string
default:: `syn-filesystem-exporter`

The namespace in which to deploy this component.
By default this is `syn-${_instance}`.

== `filesystem.path`

[horizontal]
type:: string
default:: `/data`

The mounted filesystem path scanned by the exporter. This value is also used in the generated alert expressions.

== `images.exporter`

[horizontal]
type:: object
default::
+
[source,yaml]
----
images:
  exporter:
    registry: ghcr.io
    repository: containeroo/filesystem-exporter
    tag: v1.0.0
----

The container image used for the exporter.

== `volume`

[horizontal]
type:: object
default:: `null`

The Kubernetes volume source for the single filesystem mounted by this exporter instance.
This parameter is required.

Example:
[source,yaml]
----
volume:
  persistentVolumeClaim:
    claimName: archive
----

The component hardcodes the volume name to `data`, mounts it read-only, and uses `filesystem.path` as the mount path.

== `monitoring_enabled`

[horizontal]
type:: boolean
default:: `true`

Enables the monitoring resources. When enabled, the component renders a `ServiceMonitor` and a `PrometheusRule` and adds the same namespace-level monitoring labels described by `component-backup-k8up`: `SYNMonitoring=main` on most distributions, or `openshift.io/cluster-monitoring=true` on OpenShift 4.

== `grafana_dashboard`

[horizontal]
type:: object
default::
+
[source,yaml]
----
grafana_dashboard:
  enabled: false
----

Controls an optional Grafana dashboard `ConfigMap`.

== Example

[source,yaml]
----
namespace: example-namespace
filesystem:
  path: /srv/data
volume:
  nfs:
    server: nfs.example.internal
    path: /exports/data
grafana_dashboard:
  enabled: true
----
